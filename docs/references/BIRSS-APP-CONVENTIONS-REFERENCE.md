# birss-app ‚Üî Birss Tables: Convention Contract & Verification Ladder

**Purpose.** Guarantee that, for every **Default group**, `manganite/birss-app` reproduces the Birss tables *in every respect* ‚Äî name, axes, generators, operation set, and tensor components, and specify a safe, table-anchored test for each link in the chain.

**Central app references.** This document and its companion **[`table-nomenclature.md`](./table-nomenclature.md)** (the 122-group Schoenflies / full-HM / Shubnikov / operator / generator table) are the two central convention references for the app. This document is **self-contained**: all conventions are explained here in full. It references the frozen Birss **data tables** (`table-3`, `table-4a`, `table-4e/4f`, `table-6`, `table-7`) and `table-nomenclature.md` for *data* (symbols, matrices, forms), and the Birss book for equation numbers, but no explanation is deferred to another narrative document.

**Authority.** R. R. Birss, *Symmetry and Magnetism* (1966); the frozen transcriptions in `manganite/birss-tables`. Where Birss and ITC diverge, **Birss wins** (see Step 1).

**Scope.** Default (setting-1) groups only. Birss tabulates a single setting per group; alternative axis settings (the orthorhombic setting taxonomy) are a separate concern and are explicitly out of scope here.

**Anti-circularity (non-negotiable).** Every expected value at every step is read from a Birss table (`table-3`, `table-4a`, `table-4e/4f`, `table-6`, `table-7`), never generated by running the app. A test that compares the app to itself proves nothing.

## The dependency ladder

```
(1) names ‚îÄ‚îÄ‚îê
            ‚îú‚îÄ‚îÄ‚ñ∂ (3) generators ‚îÄ‚îÄ‚ñ∂ (4) operation set ‚îÄ‚îÄ‚ñ∂ (5) tensor components

```

Steps **4 and 5 are deterministic consequences** of 1‚Äì3: group closure (4) and tensor projection (5) are algorithms with no free choices. So the correctness effort lives in **1‚Äì3**; steps 4‚Äì5 are *checks that 1‚Äì3 were right*. Crucially, **(1) and (2) are coupled** ‚Äî the canonical symbol is only defined once the axis setting is fixed (the same abstract group carries different HM symbols in different settings). Treat 1+2 as a single "identity+orientation" decision, then read generators (3) off the tables in that frame.

Each step below states: **the contract**, **the source table**, and **the test**.

## Step 1 ‚Äî Group names

### Contract
The app's group **key set** must equal the Birss symbol set for all 122 groups, in a single fixed notation variant, in the Birss axis setting (Step 2).

- **90 groups** (32 classical Type I + 58 black-white Type III): names taken from Table 3 (classical) / Table 6 (black-white), **in the app notation variant** ‚Äî identical to Birss except the cubic short-symbol convention below (app keeps `-3`/`-3'` where Birss Table 6 prints the bar-less `m3` form).
- **32 grey groups** (Type II) are **absent from Table 6** and are derived from the 32 classical by the fixed rule **grey = ùí¢ ‚äó {1, 1‚Ä≤}**, symbol = classical symbol **+ `1'`** (e.g. `mmm` ‚Üí `mmm1'`, `222` ‚Üí `2221'`). Their tensors follow trivially (Step 5: i-tensor = parent's, c-tensor ‚â° 0).

### Birss-over-ITC invariant (the one orthorhombic divergence)
The group D‚ÇÇ‚Çï(C‚ÇÇ‚Çï) ‚Äî unprimed inversion, one unprimed 2-fold + its mirror ‚Äî is named:

| Source | Symbol | un-primed 2-fold |
|---|---|---|
| **Birss (app must use this)** | **`m'm'm`** | 2_z |
| ITC Table 1.5.2.3 | `mm'm'` | 2_x |

The app **must** carry `m'm'm` (2_z) and **must not** introduce `mm'm'`. This is the only orthorhombic point where the two references disagree; because the two symbols differ by a single prime position, it is the easiest orthorhombic group to mislabel ‚Äî carrying `mm'm'` would pair the key with the wrong operator set.

### Notation-variant decision & data source
**Canonical key = abbreviated (short) Hermann‚ÄìMauguin**, in the app's notation, which *is* transcribed in the repo (table-3 / table-6), so Step-1 tests have a source of truth. App notation rules (fixed since v1, deviating from Birss where noted):
- **Time reversal = prime `'`** (not the Birss/ITC Shubnikov underline).
- **Inversion = leading `-`** (`-1`, `-4`, `-3`).
- **Cubic:** keep the inversion bar on the 3 (`-3`) and its prime where present (`-3'`), e.g. `m-3`, `m'-3'm'`, `m'-3'm` ‚Äî standard HM usage, **unlike** Birss Table 6's bare `m3` / `m'3m'`. (This is an app-vs-Birss short-symbol difference, not a group difference.)

**Data source.** Full HM and pure-magnetic-group **Schoenflies** are not in `birss-tables`; both are provided for all 122 groups in **[`table-nomenclature.md`](./table-nomenclature.md)**, transcribed from ITC Table 1.5.2.3 and joined to the app keys by abstract group `G(H)`. Notes:
- Grey (Type II) Schoenflies use the ITC **subscript R** (D<sub>4R</sub>); HM uses `‚Ä¶1'`.
- Full HM is given in the **app (Birss) setting** (monoclinic z‚à•c ‚áí `112`, not ITC b-unique `121`; D<sub>2h</sub>(C<sub>2h</sub>) ‚áí `2'/m' 2'/m' 2/m`); the ITC form is recorded per-row.

### Test
Assert `keys(GENERATORS) ‚à™ grey-derived` == `symbols(table-3) ‚à™ symbols(table-6) ‚à™ (+1' rule)`, as sets, in the chosen variant. Explicitly assert `m'm'm ‚àà keys` and `mm'm' ‚àâ keys`.

## Step 2 ‚Äî Coordinate axes / orientation

### Contract
Each Default group's reference axes follow **Birss Table 4a** ("Orientation of reference axes"), verbatim, in the app's orthonormal Cartesian frame. The app uses **pure point-group generators, no lattice/metric** ‚Äî so "axes" means the Cartesian directions assigned to the symmetry elements, nothing more.

**Per-system axis convention (Birss).** The frame `x,y,z` (= `Ox‚ÇÅ,Ox‚ÇÇ,Ox‚ÇÉ`) is right-handed orthonormal, fixed in the crystal (not the possibly-oblique `a,b,c`):
- **Triclinic:** no constraint.
- **Monoclinic:** **z** is the unique axis ‚Äî the 2-fold itself (class `2`) or the mirror normal (class `m`); Table 4a `2//z`, `-2//z`. This is Birss's **z-unique** setting; ITC's **b-unique** convention plays the same role with `b`, so ITC sources need a permutation.
- **Tetragonal / trigonal / hexagonal:** **z** = principal axis (4-, 3-, 6-fold = `c`). **y** = either a secondary 2-fold ‚ä• z (classes with `2‚ä•`, e.g. `32`, `422`, `622`; generator œÉ(2)=[2_y], Table 4a `2//y`) or a mirror normal for a plane containing z (classes `3m`, `4mm`, `6mm`; œÉ(4)=[-2_y], `-2//y`). **x** completes the right-handed frame.
- **Orthorhombic:** `x,y,z` = the three mutually ‚ä• 2-folds (or mirror normals); Table 4a `2//x, 2//y` (`2//z` implied).
- **Cubic:** `x,y,z` along the cube edges ([100], the 4- or 2-folds); the four 3-folds lie along body diagonals ([111]) and are **not** coordinate axes (œÉ(9) realizes one by cyclically permuting x‚Üíy‚Üíz).

### Load-bearing cases (where axis conventions silently diverge)
1. **Monoclinic:** Birss first setting **z ‚à• c** (y ‚à• b\*, x ‚à• a). ITC second-setting (b-unique) sources need permutation. App follows **Birss (z ‚à• c)**.
2. **Bracketed / rotated groups ‚Äî component-critical for the app.** A parenthesized symbol denotes the **same abstract group** as the unparenthesized version, but in a **non-standard axis orientation** within the crystal frame (it inherits the frame of the magnetic group it was derived from, without re-orienting to the standard HM setting). Because tensor components are tied to the fixed crystal axes, such a group **suppresses different specific components** than the standard-orientation group ‚Äî so for the app's component-level work the rotated orientation **must** be honoured. (For the coarser Table-4a *symbol-class* lookup the two are equivalent, which is why Birss's own tables treat them interchangeably there.) They are:

**Default bracketed magnetic groups** ‚Äî these *are* Default (Table A) keys; the **Table-6 operator list is the authority** for their orientation (read the axes off the actual operators, not off a naive x-y-z reading of a re-derived symbol):

| Default key | abstract group | Birss orientation (non-standard) |
|---|---|---|
| `2'm'm` | C‚ÇÇ·µ• + time reversal | 2-fold ‚à• **a** (not c); operators `1, -2_z, 2'_x, -2'_y` (Table 6) = {m_z, 2‚Ä≤_x, m‚Ä≤_y} |
| `-4'm2'` | D‚ÇÇd + time reversal | rotated setting of D‚ÇÇd |
| `-6'2m'` | D‚ÇÉh + time reversal | rotated setting of D‚ÇÉh |

**Alternate-setting pairs** (context only ‚Äî *not* Default keys, and alternate settings are out of scope here): the classical `-4m2`/`-42m` (D‚ÇÇd) differ by a **œÄ/4** rotation and `-62m`/`-6m2` (D‚ÇÉh) by **œÄ/6**. The app's Default key is `-42m` (D‚ÇÇd) and `-6m2` (D‚ÇÉh); the other member of each pair is an alternate setting, not tabulated as a Default.

### x-y-z positional rule (orthorhombic and generally)
Within an HM symbol, position 1 ‚Üí x, position 2 ‚Üí y, position 3 ‚Üí z (Birss Ch. 2). For orthorhombic groups this rule *is* the orientation, and the Default operator set must equal the x-y-z reading of the symbol. The bracketed groups are the cases where the symbol's own axis setting is non-standard, so the rule is applied to the **Birss symbol in its Birss setting** (Table 6), not to a normalized symbol.

### Test
For each group, assert the intended element‚Üíaxis assignment equals Table 4a. For the **three default bracketed groups** (`2'm'm`, `-4'm2'`, `-6'2m'`), assert the Default operator directions equal the Table-6 operator list (this subsumes the x-y-z check). The alternate-setting pairs (`-4m2`/`-42m`, `-62m`/`-6m2`) are **not** Default keys and are out of scope for this test.

## Step 3 ‚Äî Generators

### Contract
Generators are read from **Table 3** (classical, `œÉ(N)`) and **Table 6** (black-white, `œÉ(N)` unitary + `œÉ'(N)` antiunitary). The app's generator **matrices** must equal Birss's œÉ(0)‚ÄìœÉ(9) pool; antiunitary generators carry the time-reversal flag (`œÉ'`).

**The œÉ(0)‚ÄìœÉ(9) pool** (Birss eq. 2.17): ten fixed 3√ó3 matrices referenced by number in Table 3 / Table 6.

| œÉ | operation | œÉ | operation |
|---|---|---|---|
| œÉ(0) | identity `1` | œÉ(5) | mirror ‚ä• z (`-2_z` = m_z) |
| œÉ(1) | inversion `-1` | œÉ(6) | 3-fold about z (`3_z`, 120¬∞) |
| œÉ(2) | 2-fold about y (`2_y`) | œÉ(7) | 4-fold about z (`4_z`) |
| œÉ(3) | 2-fold about z (`2_z`) | œÉ(8) | roto-inversion `-4_z` |
| œÉ(4) | mirror ‚ä• y (`-2_y` = m_y) | œÉ(9) | 3-fold about [111] (cyclic x‚Üíy‚Üíz‚Üíx) |

`œÉ'(N)` = the same matrix œÉ(N) combined with time reversal (antiunitary). The explicit 3√ó3 entries are Birss eq. (2.17); Table 3's "Generating matrices" column and Table 6's œÉ(N)/œÉ'(N) columns give the per-group generator sets.

- Implement each generator as the Birss œÉ matrix, not an ad-hoc rotation, so that closure reproduces Birss's operators exactly.
- Prefer building alternate settings (out of scope here) via similarity transforms `G' = S¬∑G¬∑S‚Åª¬π` with the antiunitary flag preserved ‚Äî never by hand-writing rotated generators.

### Test
For each group, assert app generator matrices (and antiunitary flags) == the `œÉ(N)`/`œÉ'(N)` entries in Table 3 / Table 6 for that row.

## Step 4 ‚Äî Symmetry operation set

### Contract
Closing the Step-3 generators yields exactly the "Symmetry operators" column of Table 3 / Table 6, as a set, with correct unitary/antiunitary character on each element.

This step has **no free choices**: if Steps 1‚Äì3 are right, closure is forced. It is the strongest single integration test.

### Test
For all 122 groups: assert `close(app_generators)` == `operators(table-3 | table-6)` as sets (matrix identity + antiunitary flag). For grey groups, assert the operator set == parent classical set `‚à™ {g¬∑1' : g ‚àà parent}`.

## Step 5 ‚Äî Tensor components

### Contract
Given the correct operator set, the projected tensor form must equal the Birss tabulated form ‚Äî **after** applying the app's intrinsic index symmetry (particularization).

**(a) Tensor-type ‚Üí Table-7 column ‚Üí form-table map:**

| App tensor | physical tensor | Table 7 column | form table | transform law |
|---|---|---|---|---|
| **ED** | electric-dipole SHG | Polar **odd** rank n (n=3) | Table 4e | polar: `T'=RRR¬∑T` |
| **MD** | magnetic-dipole SHG | Axial **odd** rank n (n=3) | Table 4e (axial) | axial: `T'=det(R)¬∑RRR¬∑T` |
| **EQ** | electric-quadrupole SHG (output `Q_ij`, input `E_k E_l`) | Polar **even** rank m (m=4) | Table 4f | polar: `T'=RRRR¬∑T` (rank-4 polar, no det factor) |

The **i / c** flag sets the antiunitary sign: an antiunitary operation contributes **+1** for i-tensors (time-even) and **‚àí1** for c-tensors (time-odd).

**(b) Symbol class** for each (group, tensor, i/c) is read from **Table 7** (magnetic) or **Table 4a** (classical). **(c) Form** of that class is read from **Table 4e/4f**.

**(d) PARTICULARIZATION (the essential subtlety).** Tables 4a‚Äì4f give the **crystal-symmetry-only** general form ‚Äî the most general tensor consistent with the point group *alone*, before any physical (intrinsic) index symmetry. On top of it the app imposes exactly **one** intrinsic index symmetry ‚Äî **symmetry in the last two indices only** ‚Äî for every rank, because the last two indices are the two identical driving œâ-fields (`E_j E_k` for ED/MD; `E_k E_l` for EQ, with `E_{jk}=E_{kj}`). Particularization applies this on top of the table form, adding equalities among the surviving components and reducing the independent count below what the table lists.

- **ED, MD** (rank 3, `œá_{ijk}`): symmetric in `(j,k)` ‚Üí jk-symmetric.
- **EQ** (rank 4, `œá_{ij,kl}`; output pair `(i,j)` = response/gradient indices, input pair `(k,l)` = the two fields): symmetric in the **last two** `(k,l)` **only**. The output pair `(i,j)` is kept **general (all 9 components)** and is *not* symmetrized ‚Äî the EQ output is the full rank-2 object `Q_{ij}`, and only the field pair carries the two-identical-fields symmetry.

So the app's form = the table form **with the last two indices symmetrized**, never the raw table count.

- *Worked check (ED):* `mm2` ED = class **E** ‚Üí table-4e **E3 = 7** crystal-only components (zzz + xxz-family(3) + yyz-family(3)); symmetrizing `(j,k)` ‚Üí **5** independent (`œá_xxz=œá_xzx`, `œá_yyz=œá_yzy`, `œá_zxx`, `œá_zyy`, `œá_zzz`). Table lists 7, app computes 5 ‚Äî both correct in their own convention.
- *Worked check (EQ):* the `(k,l)` symmetrization identifies `œá_{ijkl}` with `œá_{ijlk}` (e.g. `œá_xyxy = œá_xyyx`), reducing the table-4f count while the `(i,j)` pair stays free; a class whose 4f row separates such permutations therefore yields *fewer* independent app components than the raw 4f entry, and the same number as 4f only when 4f already lists no `(k,l)`-distinct pair for that class.

Any Step-5 test must compare against this **last-two-symmetrized** form (ED/MD: symmetrize the 2 field indices of the table-4e row; EQ: symmetrize the last 2 indices of the table-4f row, leave `(i,j)` general).

**(e) Grey groups:** i-tensor = parent classical i-tensor; **c-tensor ‚â° 0** identically (Table 6 note). Cheap, strong check.

### Class forms actually used (rank-3, jk-particularized)
- **D‚ÇÉ** (3 indep.): `œá_xyz=œá_xzy`, `œá_yxz=œá_yzx`, `œá_zxy=œá_zyx`
- **E‚ÇÉ** (5 indep., z-unique): `œá_xxz=œá_xzx`, `œá_yyz=œá_yzy`, `œá_zxx`, `œá_zyy`, `œá_zzz` (a-/b-unique = axis-permuted)

### Test
For each (group, tensor, i/c): (1) look up the class in Table 7 / 4a; (2) look up the class form in Table 4e/4f; (3) apply the app's intrinsic symmetry (jk for ED/MD; pair for EQ); (4) assert the app's computed form equals that particularized form. Never derive the expected form from the app. Confirm grey c-tensors vanish.

## Scope & recorded decisions
- **Canonical group key** = the short Hermann‚ÄìMauguin symbol in the app's notation (Step 1). The full HM and magnetic Schoenflies notation for all 122 groups are in [`table-nomenclature.md`](./table-nomenclature.md).
- **Alternative axis settings are out of scope here** ‚Äî this document covers the Default (setting-1) group only. Settings are realized via similarity transforms (Step 3), never hand-written generators.
